# Task 001 — Fix Type-Check Errors

Source: docs/bugs/001-fix-typecheck-error.md
Primary-Topic: type-checking
Secondary-Topics: langgraph

## Key Concepts

- Running `just type-check` (basedpyright) exits with code 1 across 12 files; 50 actual errors identified, remainder are warnings that also block the check.
- Implementation plan lives at `docs/superpowers/plans/2026-06-24-fix-typecheck-errors.md`.
- Decisions were made in a grilling session dated 2026-06-24.
- D1 — Library stub gaps are fixed with targeted `# type: ignore[<code>]` rather than casts/wrappers, since these are stub gaps not real bugs. Applies to: `ChatAnthropic(model=...)` (works at runtime, stubs only know `model_name`), `Settings()` (pydantic-settings injects env vars but pyright sees missing required args), `__tablename__ = "..."` in SQLModel (stubs type it as `declared_attr[Unknown]`), `AsyncPostgresSaver(pool)` (psycopg pool type doesn't match LangGraph's `Conn` stub), `with_structured_output().ainvoke()` (returns `dict | BaseModel`, not the specific Pydantic class), presidio `RecognizerResult` list type mismatch.
- D2 — In `ingestion_agent.py`, `response.content[0].text` is unsafe; replaced with isinstance narrowing using `anthropic.types.TextBlock`: `next(b for b in response.content if isinstance(b, TextBlock)).text.strip()`. Safer than a cast — raises `StopIteration` immediately if the API ever returns a non-TextBlock first, instead of a confusing `AttributeError`.
- D3 — In `ingest.py`, `asyncio.gather(return_exceptions=True)` returns `BaseException`, not just `Exception`; changed `isinstance(result, Exception)` to `isinstance(result, BaseException)` because the former doesn't narrow `BaseException` away in the else branch.
- D4 — `ingestion_graph.py`'s `build_ingestion_graph()` return type changed from `StateGraph` to `CompiledStateGraph[IngestionState, None, IngestionState, IngestionState]` — the correct type, which also fixes a downstream `ainvoke` error in `ingest.py` for free.
- D5 — Node `dict` return types replaced with per-node output TypedDicts added to `state.py`, with no `Any` used. Each node's return type is a TypedDict matching exactly the keys it writes. TypedDicts added: `PickFileOutput` (`files: NotRequired[list[str]]`, `in_progress: str | None`), `IngestionAgentOutput` (`processed: NotRequired[list[str]]`, `in_progress: str | None`, `retry_queue: list[FailedFile]`, `failed: NotRequired[list[FailedFile]]`), `RedactInboundOutput` (`messages: list[BaseMessage]`), `RedactOutboundOutput` (`final_answer: str`), `RetrieveMemoryOutput` (`retrieved_memory: list[MemoryItem]`), `RouteQueryOutput` (`routing_decision: Literal["rag", "web", "both", "neither"]`), `RagRetrievalOutput` (`rag_results: list[RagResult]`), `WebResearchOutput` (`web_results: list[WebResult]`), `SynthesisNodeOutput` (`final_answer: str`, `confidence: float`, `is_uncertain: bool`).
- D6 — `messages[-1].content` is typed as `str | list[...]`; introduced a new util helper `get_str_content(msg: BaseMessage) -> str` in new file `apps/backend/src/second_brain/utils.py`, which raises `TypeError` (not a bare `assert`) if content is not `str`, for a clearer error message. Repeated in 4+ nodes, so centralizing it is the single change point. Placed in a dedicated `utils.py` (not `state.py`) since it's a utility, not a state definition.
- D7 — presidio `RecognizerResult` mismatch fixed with a targeted `# type: ignore[arg-type]` on the `anonymize()` call — same runtime class, different stub import paths.
- D8 — `query_graph.py` returns `tuple[CompiledStateGraph[...], AsyncConnectionPool[Any]]`; the `Any` type parameter (psycopg row-factory type) is an approved exception to the project's no-`Any` rule, since it's imposed by the library, not project code.
- D9 — `__tablename__` in `db/models.py` annotated as `ClassVar[str]` — this is the correct SQLAlchemy annotation, not a `# type: ignore` workaround.
- D10 — `ChatAnthropic(model=...)` changed to `ChatAnthropic(model_name=...)` in `orchestrator.py` and `synthesis.py`, since `model=` is a runtime-only alias invisible to the stubs.
- Full file-by-file change table ("Files to Touch"): `utils.py` (new) adds `get_str_content` helper; `graphs/state.py` adds 9 node output TypedDicts and fixes `RagResult.metadata: dict`; `services/chunking.py` changes `Chunk.metadata: dict` to `dict[str, str | int]`; `db/models.py` adds `ClassVar[str]` for all `__tablename__` and fixes bare `dict` fields; `config.py` adds `# type: ignore[call-arg]` on `Settings()`; `graphs/ingestion_graph.py` fixes return type and types `pick_file_node` as `PickFileOutput`; `graphs/query_graph.py` fixes return type and adds `# type: ignore[arg-type]` on `AsyncPostgresSaver`; `nodes/ingestion_agent.py` uses `IngestionAgentOutput` and fixes `.text` narrowing; `nodes/orchestrator.py` uses `model_name=`, `RouteQueryOutput`, and `# type: ignore[assignment]`; `nodes/synthesis.py` uses `model_name=`, `SynthesisNodeOutput`, and `# type: ignore[assignment]`; `nodes/pii_redaction.py` gets typed returns and `get_str_content`; `nodes/rag_retrieval.py` gets typed returns, `get_str_content`, and `list[RagResult]`; `nodes/web_research.py` uses `WebResearchOutput` and `get_str_content`; `nodes/memory_retrieval.py` uses `RetrieveMemoryOutput`; `services/pii.py` adds `# type: ignore[arg-type]`.
