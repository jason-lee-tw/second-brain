# Eval RAGAS Collections Migration Implementation Plan

Source: docs/superpowers/plans/2026-07-01-eval-ragas-collections-migration.md
Primary-Topic: ragas-evaluation
Secondary-Topics: dependency-management, llm-client-configuration

## Key Concepts

- Goal: fix `just eval-baseline`'s `openai.OpenAIError: Missing credentials` crash and remove `ragas.metrics`/`LangchainLLMWrapper`/`evaluate()` deprecation warnings, without breaking `just test-eval`.
- Root architecture change: add a shared `apps/eval/ragas_client.py` module providing `build_llm()`, `build_embeddings()`, and `safe_mean()`, consumed by both `baseline.py` and `run_eval.py`.
- `build_llm()` wires an Anthropic client (`anthropic.AsyncAnthropic`) through `ragas.llms.base.llm_factory(JUDGE_MODEL, provider="anthropic", client=...)` — `JUDGE_MODEL = "claude-sonnet-4-6"`.
- Known Anthropic quirk baked into `build_llm()`: `claude-sonnet-4-6` rejects `temperature` + `top_p` together (HTTP 400); RAGAS's `InstructorModelArgs` defaults both, so the code explicitly pops `top_p` from `llm.model_args` while keeping `temperature`. This fix (plus switching to async clients) was added after Task 1 originally shipped, discovered during Task 5's live verification (see design doc Decisions log #8-9, commits `63c7ed9`/`435619e`).
- `build_embeddings()` points an OpenAI-compatible client at local Ollama (`OLLAMA_URL = http://localhost:11434`, `/v1` suffix, dummy `api_key="ollama"`) through `ragas.embeddings.base.embedding_factory("openai", model=EMBEDDING_MODEL, client=...)`. `EMBEDDING_MODEL = "qwen3-embedding:0.6b"`.
- `safe_mean(values: list[float]) -> float | None` — NaN-tolerant averaging: filters out `nan` values, rounds to 4 decimal places, returns `None` if the list is empty or every value is NaN. Replaces RAGAS's old `evaluate(raise_exceptions=False)` behavior of silently excluding failed samples from the mean.
- `score_or_nan(metric, **kwargs) -> float` — added later during code review (not part of Task 1's original scope, see Decisions log #10); wraps a single `.ascore()` call, converting exceptions into NaN so one failing sample doesn't crash the whole batch or lose other samples' scores. Both `baseline.py` and `run_eval.py` import it.
- Migration replaces `ragas.evaluate()` / `EvaluationDataset` / `SingleTurnSample` / `LangchainLLMWrapper` with a hand-written `async def _score_all(results)` loop per file that calls `ragas.metrics.collections.{Faithfulness, AnswerRelevancy, ContextPrecision, ContextRecall}.ascore()` per sample, wrapped in `asyncio.run()` inside the existing synchronous `compute_baseline_metrics()`/`compute_rag_metrics()` functions — public signatures for both stay `(results: list[dict]) -> dict`, unchanged.
- Tech stack: Python 3.12, `ragas` pinned at `>=0.2.0,<0.5` (0.4.3 already ships `ragas.metrics.collections`, no version bump needed), `anthropic`, `openai` (already transitively installed via ragas; talks to local Ollama, not api.openai.com), `instructor` (transitively installed), pytest + `unittest.mock`.
- No new dependencies are added anywhere in this plan.
- `langchain-anthropic` and `pandas` become fully unused after the migration and must be removed via `uv remove --directory apps/eval langchain-anthropic pandas` — never hand-edit `uv.lock`. This is explicitly flagged per the project's CLAUDE.md rule that dependency changes must be surfaced, not silently applied.
- Per project CLAUDE.md "Done Means": `just lint`, `just type-check`, and `just test-eval` must all pass after every task; TDD (failing test → implementation → passing test) required for every behavior change.
- Task 1 — `ragas_client.py`: write failing tests in `apps/eval/tests/unit/test_ragas_client.py` covering `safe_mean` (averages, NaN-exclusion, all-NaN→None, empty→None, 4-decimal rounding), `build_llm` (asserts `anthropic.Anthropic`/`llm_factory` called with `JUDGE_MODEL`, `provider="anthropic"`), and `build_embeddings` (asserts `openai.OpenAI` pointed at `{OLLAMA_URL}/v1` with `api_key="ollama"`, and `embedding_factory("openai", model=EMBEDDING_MODEL, client=...)`). Implementation module holds constants `ANTHROPIC_API_KEY`, `OLLAMA_URL`, `EMBEDDING_MODEL`, `JUDGE_MODEL`.
- Task 2 — migrate `baseline.py`: drop pandas-based mock helper `_mock_ragas_result` and `import pandas as pd` in favor of a `_mock_metric(value)` helper returning a `MagicMock` whose `.ascore` is an `AsyncMock` returning `ragas.metrics.result.MetricResult(value=...)`. `compute_baseline_metrics` now only returns `faithfulness` and `answer_relevancy` (no `context_recall`/`context_precision` — baseline has no retrieval). Baseline's `_score_all` uses the sample's own `expected_answer` as a proxy `retrieved_contexts` for the Faithfulness metric (flagged inline with a `# ponytail:` comment as a deliberate proxy-for-no-retrieval shortcut) — this measures whether the no-RAG answer is consistent with ground truth. A metric exception on one sample must not lose other samples' scores (verified via a test where `.ascore()` raises on the first call and succeeds on the second).
- Task 3 — migrate `run_eval.py`: same pattern as Task 2 but for all four RAGAS metrics (`ContextRecall`, `ContextPrecision`, `Faithfulness`, `AnswerRelevancy`) since RAG results include real `retrieved_contexts`. `ContextRecall`/`ContextPrecision` take `reference=expected_answer` + `retrieved_contexts`; `Faithfulness` takes `retrieved_contexts` (the real ones, not a proxy); `AnswerRelevancy` only needs `user_input`/`response`. `test_smoke.py` is updated in both Task 2 and Task 3 to drop `_mock_ragas_result`/`pandas` and use the shared `_mock_metric` helper plus patched `build_llm`/`build_embeddings`.
- Task 4 — dependency removal: confirm via `grep -rn "langchain_anthropic\|^import pandas\|import pandas as" apps/eval --include="*.py"` that nothing still imports them, then `uv remove --directory apps/eval langchain-anthropic pandas`, then `uv sync --all-extras && just test-eval` to confirm the workspace still resolves.
- Task 5 — verification: `just lint`, `just type-check`, then live end-to-end checks — `just up-all` (Ollama/Postgres/Phoenix/backend healthy), confirm `qwen3-embedding:0.6b` is pulled in Ollama (`curl -s http://localhost:11434/api/tags | grep -o 'qwen3-embedding[^"]*'`, else `ollama pull qwen3-embedding:0.6b`), run `just eval-baseline` (expect no deprecation warnings, no `openai.OpenAIError`, exit 0, non-null faithfulness/answer_relevancy scores, writes `apps/eval/results/baseline.json`), run `just eval-rag` (same but all four metrics, writes `apps/eval/results/rag.json`), run `just eval-report` (prints markdown comparison table), and a final `just test-eval && just lint && just type-check` full-suite check.
- File/module map: `apps/eval/ragas_client.py` (new shared helper), `apps/eval/baseline.py` (no-RAG baseline eval), `apps/eval/run_eval.py` (RAG pipeline eval, also does `call_query_endpoint`/`embed_query`/`fetch_top_k_chunks`/`run_rag_eval` against `/query`, Ollama embeddings, and Postgres via `psycopg`), `apps/eval/pyproject.toml`/`uv.lock` (dependency removal), `apps/eval/tests/unit/{test_ragas_client,test_baseline,test_run_eval,test_smoke}.py`.
- Each task ends with its own git commit using Conventional Commits format, e.g. `feat(eval): add shared ragas_client helper for LLM/embeddings setup`, `fix(eval): migrate baseline.py to ragas.metrics.collections`, `fix(eval): migrate run_eval.py to ragas.metrics.collections`, `chore(eval): remove langchain-anthropic and pandas, unused after RAGAS migration`.
- Plan instructs agentic workers to use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` skill to implement task-by-task, tracked via checkbox syntax.
