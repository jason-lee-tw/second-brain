# Fix Type-Check Errors Implementation Plan

Source: docs/superpowers/plans/2026-06-24-fix-typecheck-errors.md
Primary-Topic: type-checking
Secondary-Topics: langgraph-state-typing, third-party-stub-gaps

## Key Concepts

- Goal: make `just type-check` exit 0 by fixing all 50 basedpyright errors across 12 backend files, with a hard constraint of never using `Any` in code the team owns.
- Three fix categories: (1) targeted `# type: ignore[<code>]` comments for third-party library stub gaps where runtime behavior is correct but stubs disagree; (2) proper `TypedDict`s for every LangGraph node's return type in `state.py`, replacing bare `dict` returns; (3) a new `second_brain/utils.py` module with `get_str_content` to narrow `BaseMessage.content` (typed as `str | list`) down to `str`, raising `TypeError` if it isn't a string.
- Design rationale and constraints (D1–D10) live in `docs/bugs/001-fix-typecheck-error.md` — referenced as the authoritative decision record for "why" each fix category was chosen.
- Global constraints applied after every task: `just type-check` must exit 0 (at the end), `just test-unit` must stay green, `just lint` must pass, no `Any` in owned code (`cast(SomeType, ...)` allowed at system boundaries like DB reads), every targeted `# type: ignore` must include the specific error code (e.g. `# type: ignore[call-arg]`).
- Tests live under `apps/backend/tests/unit/` mirroring the source tree; run via `uv run --package second-brain pytest apps/backend/tests/unit -v` (whole suite) or targeting one file.
- Task 1 — `get_str_content(msg: BaseMessage) -> str` in new `apps/backend/src/second_brain/utils.py`: raises `TypeError` with a message naming the actual type (e.g. "Expected str content, got list") when `msg.content` is not a `str`. Exists because LangChain types `BaseMessage.content` as `str | list[...]` for multi-modal support, but this app only ever sends plain text.
- Task 2 — new TypedDicts added to `apps/backend/src/second_brain/graphs/state.py` after the existing `CorrectionUpdate`:
  - Ingestion graph outputs: `PickFileOutput` (`TypedDict, total=False` with `in_progress: str | None` and `files: NotRequired[list[str]]`, because Python can't cleanly mix required + `NotRequired` under `total=False`, and `total=False` matches LangGraph's partial-state-update merge semantics), `IngestionAgentOutput` (`in_progress`, `retry_queue: list[FailedFile]`, optional `processed`/`failed`).
  - Query graph outputs: `RedactInboundOutput` (`messages: list[BaseMessage]`), `RedactOutboundOutput` (`final_answer: str`), `RetrieveMemoryOutput` (`retrieved_memory: list[MemoryItem]`), `RouteQueryOutput` (`routing_decision: Literal["rag", "web", "both", "neither"]`), `RagRetrievalOutput` (`rag_results: list[RagResult]`), `WebResearchOutput` (`web_results: list[WebResult]`), `SynthesisNodeOutput` (`final_answer: str`, `confidence: float`, `is_uncertain: bool`).
  - `RagResult.metadata` changed from bare `dict` to `dict[str, str | int]`.
  - New `ChunkMetadata` TypedDict added to `apps/backend/src/second_brain/services/chunking.py` with keys `source: str`, `heading_path: str`, `content_type: str`, `char_count: int`; `Chunk.metadata` field retyped from bare `dict` to `ChunkMetadata`.
  - Needs `from langchain_core.messages import BaseMessage` import added to `state.py`; `NotRequired` and `Literal` already imported there.
- Task 3 — pure annotation fixes, no behavior change, across three files:
  - `db/models.py`: every SQLModel `__tablename__` attribute annotated `ClassVar[str]` (needs `ClassVar` import) — covers `ChatHistory` (`chat_history`), `IngestedDocument` (`ingested_documents`), `DocumentChunk` (`document_chunks`), `LearnedFact` (`learned_facts`), `ModelCorrection` (`model_corrections`). Also fixes bare `dict` fields: `ChatHistory.thread_data` becomes `dict[str, object]` (LangGraph manages the structure), `DocumentChunk.chunk_metadata` becomes `Optional[dict[str, str | int]]` (mapped to SQL column `metadata` via `sa_column=Column("metadata", ...)`, per the project's `chunk_metadata`/`metadata` naming split to avoid a SQLAlchemy conflict).
  - `config.py`: `settings = Settings()` gets `# type: ignore[call-arg]` — a stub gap for pydantic-settings where required fields are populated from env vars at runtime but the stub can't see that.
  - `services/pii.py`: `_anonymizer.anonymize(...)` call gets `# type: ignore[arg-type]` — a presidio stub gap (`RecognizerResult` typing mismatch, D7).
- Task 4 — graph builder return types in `graphs/ingestion_graph.py` and `graphs/query_graph.py`:
  - `pick_file_node` retyped to return `PickFileOutput` instead of bare `dict`.
  - `build_ingestion_graph() -> CompiledStateGraph[IngestionState, None, IngestionState, IngestionState]` (was untyped/`StateGraph`).
  - `build_query_graph(postgres_url: str) -> tuple[CompiledStateGraph[SecondBrainState, None, SecondBrainState, SecondBrainState], AsyncConnectionPool[Any]]` — the `Any` here is explicitly excepted from the "no Any" rule because it's psycopg's row-factory type parameter, not code the team owns (marked with a `ponytail:` comment).
  - `AsyncPostgresSaver(pool)` call gets `# type: ignore[arg-type]` (D8: LangGraph checkpointer stub gap).
- Task 5 — `nodes/ingestion_agent.py`: narrows an Anthropic SDK response content block to `TextBlock` before accessing `.text` (`next(b for b in response.content if isinstance(b, TextBlock))` instead of blind `response.content[0].text`), since `response.content` is a union of block types (D2). Also retypes `ingestion_agent_node(state: IngestionState) -> IngestionAgentOutput` (body unchanged — the three existing return literals already match the TypedDict shape).
- Task 6 — `nodes/orchestrator.py` and `nodes/synthesis.py`: `ChatAnthropic(model=...)` renamed to `ChatAnthropic(model_name=...)` because the stub only exposes `model_name` as a typed kwarg (D10, applies to both files). `route_query` retyped to return `RouteQueryOutput`; `synthesize_answer` retyped to return `SynthesisNodeOutput`. Both structured-output `.ainvoke(prompt)` results get `# type: ignore[assignment]` because `with_structured_output` returns a stub type broader than the specific Pydantic model assigned.
- Task 7 — applies `get_str_content` at every site that reads `messages[-1].content` directly, and fixes an exception-narrowing bug:
  - `pii_redaction.py`: `redact_inbound` raises `ValueError` if `state["messages"]` is empty (new guard), then uses `get_str_content(last)` before passing to `redact_pii`; returns `RedactInboundOutput`. `redact_outbound` returns `RedactOutboundOutput`.
  - `rag_retrieval.py`: `_query_pgvector` return type becomes `list[RagResult]`, with the metadata dict wrapped in `cast("dict[str, str | int]", ...)` at the DB-read boundary (an explicitly sanctioned `Any`-free use of `cast`). `retrieve_from_rag` uses `get_str_content(state["messages"][-1])` for the query text and returns `RagRetrievalOutput`.
  - `web_research.py`: `search_web` uses `get_str_content` for the query and returns `WebResearchOutput`.
  - `memory_retrieval.py`: `retrieve_memory` retyped to return `RetrieveMemoryOutput`.
  - `api/routers/ingest.py`: bug fix — `isinstance(result, Exception)` changed to `isinstance(result, BaseException)` (D3) when checking `asyncio.gather` results, since gathered task exceptions can be `BaseException` subclasses not caught by the narrower `Exception` check.
  - This task's final type-check pass is called out as "the milestone" — the point where `just type-check` should exit 0 for the first time.
- Task 8 — final verification: `just lint && just format && just type-check` all exit 0 (format produces no diff or the diff gets committed); full `just test-unit` passes with no regressions; manually scan type-check output for stray `error:`/`warning:` lines; run the `enhanced-review` skill on the diff before opening the PR; push branch `fix/typecheck-errors` and open a PR (title `fix(types): resolve all 50 basedpyright errors`) summarizing all the above changes.
- Self-review table maps each of the 10 design decisions (D1–D10) from `docs/bugs/001-fix-typecheck-error.md` to the task(s) that implement it, confirming full coverage: D1 stub-gap ignores → Tasks 3/4/6; D2 TextBlock narrowing → Task 5; D3 `BaseException` isinstance → Task 7; D4 `CompiledStateGraph` return type → Task 4; D5 node output TypedDicts → Task 2; D6 `get_str_content` → Tasks 1/7; D7 presidio `RecognizerResult` ignore → Task 3; D8 `AsyncConnectionPool[Any]` exception → Task 4; D9 `ClassVar[str]` for `__tablename__` → Task 3; D10 `model_name=` in ChatAnthropic → Task 6.
