# RAGAS Collections Migration

Migrates `apps/eval`'s RAGAS scoring off the deprecated `evaluate()`/`EvaluationDataset` API onto `ragas.metrics.collections`, fixing a live `OpenAIError: Missing credentials` crash while keeping `compute_baseline_metrics()`/`compute_rag_metrics()` signatures unchanged.

## Key Concepts

- **Problem**: `just eval-baseline` crashed with `openai.OpenAIError: Missing credentials` and printed `ragas.metrics`/`LangchainLLMWrapper` deprecation warnings.
- **Root cause**: not the judge LLM — both files already used Anthropic (`ChatAnthropic(model="claude-sonnet-4-6")`) for judging. `AnswerRelevancy` also needs a separate **embeddings** model (cosine similarity between the original question and one regenerated from the answer); neither file passed embeddings explicitly, so RAGAS's `evaluate()` fell back to its OpenAI default embedder, which requires `OPENAI_API_KEY`. Anthropic/xAI don't expose public embeddings APIs, so swapping the judge provider would not have fixed it.
- **Why not a one-line fix**: passing `embeddings=` into the existing `evaluate()`/`EvaluationDataset` call was ruled out because `ragas.metrics.{AnswerRelevancy,Faithfulness}` (legacy) are deprecated in favor of `ragas.metrics.collections.{AnswerRelevancy,Faithfulness}`, which inherit from a different base class (`collections.base.BaseMetric` vs `metrics.base.Metric`) that `ragas.evaluate()`'s `isinstance(m, Metric)` check rejects outright. `evaluate()` itself also emits its own deprecation warning. Conclusion: drop `evaluate()`/`EvaluationDataset`/`SingleTurnSample`/`LangchainLLMWrapper` entirely and hand-write the scoring loop against the collections metrics' async `.ascore()` API.
- **New shared module `apps/eval/ragas_client.py`**, consumed by both `baseline.py` and `run_eval.py`:
  - `build_llm()` — Instructor-based Anthropic LLM via `ragas.llms.base.llm_factory(JUDGE_MODEL, provider="anthropic", client=anthropic.AsyncAnthropic(...))`, `JUDGE_MODEL = "claude-sonnet-4-6"`. Pops `top_p` from `llm.model_args` after the factory returns it, since `claude-sonnet-4-6` rejects `temperature` + `top_p` both being set (HTTP 400) and RAGAS's `InstructorModelArgs` defaults both.
  - `build_embeddings()` — local Ollama embeddings via `ragas.embeddings.base.embedding_factory("openai", model=EMBEDDING_MODEL, client=openai.AsyncOpenAI(base_url=f"{OLLAMA_URL}/v1", api_key="ollama"))`, `EMBEDDING_MODEL = "qwen3-embedding:0.6b"` — reuses the same Ollama model already used for RAG retrieval, no OpenAI key needed.
  - Both clients must be **async** (`AsyncAnthropic`/`AsyncOpenAI`), not sync — collections metrics call `agenerate()`/`aembed_text()` internally; sync clients raise `TypeError: Cannot use agenerate() with a synchronous client`. Discovered live during verification, not anticipated in the original design.
  - `safe_mean(values: list[float]) -> float | None` — NaN-tolerant averaging: filters `nan`, rounds to 4 decimals, returns `None` if the list is empty or all-NaN. Replaces `evaluate(raise_exceptions=False)`'s old silent-exclusion behavior.
  - `score_or_nan(metric, **kwargs) -> float` — centralizes per-sample scoring try/except: awaits `metric.ascore(**kwargs)`, returns `.value`; on any exception logs the metric class name + exception to stderr and returns `float("nan")` so one bad sample doesn't lose the rest of the batch. Added during code review to replace 6 duplicated inline try/except blocks (2 in `baseline.py`, 4 in `run_eval.py`) that silently swallowed errors with no logging.
- **`baseline.py`**: `_score_all(results)` builds `llm`/`embeddings` once, constructs `Faithfulness(llm=llm)` and `AnswerRelevancy(llm=llm, embeddings=embeddings)`, and scores sequentially. Only these two metrics apply — no retrieval step exists in the no-RAG baseline for `context_recall`/`context_precision`. Faithfulness uses `retrieved_contexts=[result["expected_answer"]]` as a `# ponytail:`-marked proxy for the missing retrieval context. `compute_baseline_metrics(results)` keeps its sync signature, wrapping `asyncio.run(_score_all(results))` then applying `safe_mean` per metric.
- **`run_eval.py`**: same pattern extended to all four collections metrics — `ContextRecall`, `ContextPrecision`, `Faithfulness`, `AnswerRelevancy` — scored against the pipeline's real `retrieved_contexts` (no proxy needed). Keeps its own separate `OllamaEmbeddings` (via `langchain-ollama`) for embedding the *query* in the pgvector similarity search — a distinct consumer from the RAGAS judge's embeddings client; only the metric construction switches to `ragas_client`.
- **Dependency cleanup**: `langchain-anthropic` and `pandas` become fully unused after the migration and are removed via `uv remove --directory apps/eval langchain-anthropic pandas` (never hand-edit `uv.lock`), flagged per the project's CLAUDE.md rule that dependency changes must be surfaced. No new dependencies added — `ragas` stays pinned at `>=0.2.0,<0.5` since 0.4.3 already ships `ragas.metrics.collections`.
- **Live verification findings**: initial `just eval-baseline`/`just eval-rag` runs exited 0 but every metric was `null` despite green tests/lint/type-check — root cause was `_score_all()`'s bare `except Exception: append(nan)` silently swallowing real errors the mocked unit tests never exercised. Two real bugs found and fixed live: (1) sync clients raised `TypeError: Cannot use agenerate() with a synchronous client` — fixed by switching to `AsyncAnthropic`/`AsyncOpenAI`; (2) Anthropic HTTP 400 on `temperature`+`top_p` both set — fixed by popping `top_p` from `llm.model_args`. Final state: no `OpenAIError`, no deprecation warnings, non-null scores for all metrics in both baseline and RAG eval.
- **Testing**: strict TDD — `apps/eval/tests/unit/{test_ragas_client,test_baseline,test_run_eval,test_smoke}.py` mock `build_llm`/`build_embeddings` as sentinels and mock each collections metric class so `.ascore()` is an `AsyncMock` returning a fixed `.value`, plus a test for the per-sample-failure → NaN → excluded-from-mean path. `just test-eval` stayed green throughout — no red-then-fix-later.
- **What did not change**: `generate_dataset.py`, `compare.py`, the answer-generation loops in `run_baseline()`/`run_rag_eval()`, public function signatures (`compute_baseline_metrics`, `compute_rag_metrics`, `main()`), and `.env.template` (no new env vars — `OLLAMA_URL` already existed).
